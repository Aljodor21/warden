Esto solo describió tu app — todavía faltan pasos en tu repo y en GitHub para que el despliegue automático funcione de verdad. Hacelos en orden.

0. Tu repo necesita un Dockerfile y un docker-compose.yml (en la raíz)

Si tu app YA tiene estos archivos, revisá que coincidan con esto — son los 2 puntos que warden necesita para poder gestionarla (el nombre del contenedor y el puerto):

services:
  {{.Tag}}:
    build: .
    container_name: {{.Container}}        # debe ser EXACTO a lo que pusiste en "Contenedor principal"
    restart: unless-stopped
    ports:
      - "{{.Port}}:{{.Port}}"              # el de la izquierda debe ser igual al "Puerto" que pusiste
                                            # (el de la derecha es el puerto REAL dentro de tu app —
                                            # pueden ser distintos, ajustalo a lo que tu app use)

Si todavía no tenés Dockerfile, el patrón general es siempre el mismo (cambia el comando exacto según tu lenguaje): instalar dependencias → copiar el código → exponer el puerto → definir cómo arranca. Ejemplo con Node (lo mismo aplica con otras bases para Python, .NET, etc.):

FROM node:22-alpine
WORKDIR /app
COPY package.json .
RUN npm install --omit=dev
COPY . .
EXPOSE {{.Port}}
CMD ["node", "server.js"]

1. Agregar el workflow al repo de tu app

En GitHub, abrí tu repo ({{.Install}}) → botón "Add file" → "Create new file" → nombre del archivo: .github/workflows/deploy.yml → pegá esto → "Commit new file":

name: deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: [self-hosted, warden]
    steps:
      - uses: actions/checkout@v4

      - name: Construir y levantar
        run: |
          docker compose build
          docker compose up -d

      - name: Publicar en Cloudflare
        run: sudo /usr/local/bin/warden publish

2. Registrar el runner (una sola vez por repo)

En GitHub: tu repo → Settings → Actions → Runners → "New self-hosted runner" → Linux. Te va a mostrar un token (después de --token) — copialo.

Por consola, en este server:

sudo warden runner {{.Install}} TU_TOKEN_AQUI

3. Hacer push

Cualquier git push a la rama main de ese repo va a disparar el despliegue solo, de ahora en adelante. La app va a aparecer en el Dashboard y el Catálogo cuando termine.
